## 参考答案（做完再看）

**A1.** 普通调 LLM 是「一问一答、一次性」：给 prompt、拿一段文字。Agent 是给一个
**目标**，模型自己**多步决策、调用工具、根据中间结果决定下一步**，在一个循环里
反复进行直到完成。核心区别是**循环 + 工具 + 自主决策**。

**A2.** **错。** LLM 本身只会输出文字，它并不会真的连数据库。它只是输出一段结构化
请求（如 `{"tool":"查订单","args":{...}}`），**由你写的代码**去真正执行查询，再把
结果喂回给模型。模型与真实世界之间永远隔着你的一层代码。

**A3.** LLM（大脑，负责推理决策）、Tools（手脚，负责真正做事）、Memory（记忆，记住
对话历史与事实）、Loop（循环，反复决策→行动→观察）。客服例子：理解用户在问退货 /
查订单数据库 / 记住上一句的订单号 / 查完订单再查物流。

**A4.** **一次 LLM 调用**即可。翻译是固定的、一步到位的任务，步骤数确定、不需要调用
外部工具、不需要根据中间结果动态决策，搭 agent 是过度设计。

**A5.**（示例，合理即可）
- 第 1 轮：**想**——模型发现要知道「什么时候到」得先拿到订单，但用户没给订单号，
  于是决定先调用「按用户查订单」工具。**做**——代码查库返回订单 123。**看**——拿到了
  订单，但里面只有下单信息，没有物流状态。
- 第 2 轮：**想**——需要物流状态，决定调用「查物流」工具，参数是订单 123 的运单号。
  **做**——查物流返回「预计明天下午送达」。**看**——信息够了。
- 第 3 轮：**想**——模型判断已能回答，输出最终答复「您的鞋预计明天下午送达」，跳出循环。
