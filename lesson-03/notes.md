# L3 讲义:tool use 深入

> 在 L2 单工具 loop 的基础上往深走。核心发现:loop 一行不用改,深度全在工具侧。

## 0. 本节目标
- 多工具时模型如何**选对工具**(description = 路由依据)
- **错误也是信息**:工具报错回传模型,agent 自愈
- **链式调用**:串行多步是 loop 白送的,零新代码
- 实现**循环次数上限**兜底(学习者 L2 quiz 自己提出的)

## 1. 多工具:模型怎么选?
靠读每个工具的 `description` 做意图匹配。工具越多,description 质量越生死攸关——两个工具描述含糊重叠,模型就会选错。

写好 description 三要点:
- 说清**做什么**:「根据订单号查询物流状态」≠「查订单」
- 说清**何时用**:「用户想退款/退货时调用」
- 说清**何时别用**:「仅适用于已签收的订单」——业务规则前置,减少无效调用

## 2. 错误也是信息,要回传给模型(本节最重要的思想)
> 工具报错 ≠ agent 失败。把错误作为 tool 结果回传,模型会自己调整策略:换参数重试、改调别的工具、或体面地告诉用户。

```
坏:工具抛异常 → Python 崩溃 → 用户看到 Traceback          (agent 死了)
好:异常被 except 捕获 → 错误文本作为 tool 结果回传 → 模型自己想办法 (agent 自愈)
```

模型的「自愈」是白送的,你只需保证**错误能以文字到达它眼前**。工具返回值统一用 JSON 字符串:成功和失败走同一条通道。

## 3. 链式调用:loop 白送的超能力
「帮我退掉 A123」→ 先查状态 → 再决定退不退。第二步依赖第一步。

需要的新代码:**零**。模型第一轮点单 `query_order`,第二轮看到结果自己决定下一步。`while` 循环天然支持任意长的工具链——多轮串行编排是**涌现**的,不是额外实现的。

区分两种多工具调用:
- **并行**:一轮点多个单(L2 的 A123+B456 同时查),适合互不依赖的查询
- **链式(串行)**:跨轮次,后一步依赖前一步结果。模型自己判断用哪种。

## 4. 兜底:循环次数上限
模型可能抽风一直点单。`while True` 改为有限循环(如最多 8 轮),超限优雅退出而不是烧 token。生产必备。(学习者在 L2 quiz Q4 自己提出了这个方案,本节亲手实现。)

## 5. 作业
`homework/refund_agent.py`:退款客服 agent。**loop 直接给出**(已有肌肉),挖空 L3 新肌肉:
1. 从零写 `apply_refund` 的 TOOLS schema(含让模型「先查再退」的 description 措辞)
2. try/except 包工具执行,异常 → 错误 JSON 回传
3. 实现循环次数上限

跑通标准:
- 「退掉 B456(已签收)」→ 链式:查 → 退 → 成功
- 「退掉 A123(运输中)」→ 查 → 模型解释不能退(或退款工具拒绝)
- 「退掉 C999(不存在)」→ 错误回传 → 模型体面告知,不崩溃
