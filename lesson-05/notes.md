# L5 讲义:记忆与状态(阶段二第一课)

> 接 L4 capstone:你写的累积 messages 就是"短期记忆"。本节划清记忆的边界——
> 短期 vs 长期 vs 状态,并实现最朴素的长期记忆(存文件 + 下次注入)。

## 0. 你已经写过记忆了
capstone 里 `main` 维护的贯穿会话 `messages` = **短期记忆 / 工作记忆**。"把它退了"认得"它",
就是它的功劳。也回扣 L2 学习者自己追问出的:"LLM 无持续连接、追问会忘、messages 越滚越长"。

## 1. 短期记忆活不过重启
`messages` 是内存里的 Python 列表,进程一死就没。查完 B456 关掉程序再开,agent 就失忆。
→ 这是 L5 要补的缺口。

## 2. 三个必须分清的概念
| 概念 | 是什么 | capstone 例子 | 活多久 |
|---|---|---|---|
| 短期记忆 | 当前对话的 messages | 累积的对话历史 | 一次会话/进程 |
| 长期记忆 | 跨会话记住、存在 messages 之外 | (还没有)用户名/偏好/历史退款 | 永久(落盘/入库) |
| 状态 state | 程序自己维护的事实,不一定给模型看 | REFUNDED 集合、POINTS 字典 | 看存哪 |

**记忆 vs 状态**:记忆是"塞进 messages 给模型读的";状态是"代码逻辑自己用的"。
REFUNDED 就是纯状态——防重复退款靠 `if order_id in REFUNDED`,根本没进模型的眼睛。
一句话:**记忆给模型看,状态给代码用**(有时同一份数据两边都要)。

## 3. 长期记忆 = "存 + 取"读写闭环
短期记忆是 messages 自动累积、白送的;长期记忆得自己实现两个动作:
- **存(write)**:会话中/结束时,把值得记住的写到持久层(文件/DB/向量库)。
- **取(read)**:新会话开始时,读出相关记忆,注入 messages(通常放进 system prompt)。

心智模型:**模型永远无状态(L2 就懂)。所谓"记得",全是每次开会话把该记的重新喂回它眼前。**
没有魔法,就是"存下来 + 下次读出来拼进 prompt"。

## 4. 两个设计难题(埋 L6/L7)
- **记什么/存多少**:不能每句都永久记,messages 会爆、token 会炸 → 筛选+摘要。对话太长怎么压缩 = L6。
- **怎么取相关的**:用户档案几行全量注入没问题;历史对话几百轮不能全塞 → 按相关性检索 = L7 RAG
  (思路同学习者早想到的 tool RAG)。

L5 先掌握最朴素版:存 JSON 文件 + 全量注入 system prompt。L6/L7 再教规模化。

## 5. 作业(见 homework/agent_with_memory.py)
给 capstone agent 加最朴素的长期记忆:
- load_memory()/save_memory():读写一个 memory.json
- 会话启动时把长期记忆注入 system prompt
- 会话中识别到"值得记住的事实"(用户名/偏好)时写回文件
跑通标准:会话1 告诉它"我叫小明,只收顺丰"→ 退出 → 会话2 重启后它还记得。
