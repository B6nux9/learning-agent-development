## 三、三个重大发现

**1. 已学的正是最高频的。** 工具调用 10/10、记忆 8/10 —— L2~L5 的地基完全打在刀刃上,无浪费。

**2. 大纲三处错位 + 三处缺失。**
- **错位**:框架(6/10)、Multi-Agent(5/10)被扔进 L13-14「拓展」,但 JD 里是主线要求,
  且"用过 LangChain 吗"是简历筛选与面试的直接提问 → **大幅前移**。
- **缺失**:任务规划/ReAct(6/10)、MCP(4/10)、后端工程(7/10) → 前两个必补,后端见决策 1。

**3. 被低估的加分项**:拼多多把"熟练使用 Cursor/Claude Code、具备 AI 原生思维"写进**硬性要求**,
沃孚更是核心门槛。**学习者本人正在用 Claude Code 全程学 agent 开发 —— 这本身就是简历素材和
面试话题**(可讲:如何用 agent 工具学 agent 开发、对模型行为的判断力)。

## 四、两个战略决策(已与学习者确认)

**决策 1 —— 后端工程(7/10)怎么办?** → **只补"够项目用"的最小量**。
用 FastAPI 把 agent 包成 HTTP 服务 + Docker 打包,够简历写"完整落地"、面试能讲。
**不碰**高并发/分布式深水区(1-3 个月内两头都不深,不划算)。

**决策 2 —— 框架学到什么程度?** → **用 LangChain 重写一遍现有项目**。
手写打底 + 框架落地,两头都占。面试可讲:"我知道框架底下是什么,因为我手写过" —— 强加分回答。

## 五、调整后的大纲

### ✅ 阶段一(已完成)
L1 什么是 agent · L2 手写 agent loop · L3 tool use 深入 · L4 多工具编排
→ capstone:命令行多工具客服 agent ✅

### 🔵 阶段二:核心能力补齐(投递前必做)
- **L5 记忆与状态** ✅ 已完成
- **L6 上下文管理 + Prompt/Context Engineering**(4/10 + 5/10,**合并**;沃孚 JD 就是把
  Prompt/Context/Harness Engineering 并列的)
- **L7 RAG 与向量数据库**(**8/10,第二高频,重中之重**)
- **L8 任务规划:ReAct 与 Plan-and-Execute**(6/10,**新增**)
- **L9 主流框架:用 LangChain 重写项目**(6/10,**从 L14 前移**;简历关键词)
- **L10 MCP 协议**(4/10,**新增**;2025-26 热点,面试常问)
- 🎯 **阶段二 capstone:简历级客服 Agent 项目**(最重要产出,直接写进简历)

### 🟢 阶段三:工程化与求职冲刺(投递前做完)
- **L11 Multi-Agent 多智能体协作**(5/10,**从 L13 前移**;是 L4 编排的自然延伸)
- **L12 评估体系**(对标蔚蓝 JD:工具调用准确率、任务完成率、响应延迟)
- **L13 可观测性、成本与延迟**(对标拼多多 JD:成功率、延迟、Token 消耗、业务转化率)
- **L14 服务化与部署:FastAPI + Docker**(决策 1 的最小量;含 Agent 安全基础:
  权限控制、输出过滤、高危操作二次确认)
- 🎯 **阶段三 capstone:项目上线 + 简历/面试包装**

### ⚪ 长期作战(第一轮投递后继续)
高并发与分布式深化 · 模型微调(SFT/RLHF,多为加分项) · 多模态 ·
KV Cache/vLLM 推理优化 · Agent 安全深水区(沙箱执行) · 前沿跟踪

### 砍掉/降级的内容
- 模型微调、多模态、推理优化 → 多为加分项,1-3 个月内投产比低,**降为概念级了解**。
- 原「L12 打磨」「L15 前沿」→ 并入阶段三 capstone 与长期作战。
